# Privacy Policy

Version: 1.1  
Effective Date: April 21, 2026

## 1. Data Processing Principles
Chimera adheres to the principles of **Data Minimization** and **Local Processing**. The core design goal of this application is to provide utility services without accessing user privacy.

## 2. Zero Data Collection
This application does not have any form of remote data collection, storage, or transmission capabilities.
- **Personal Information**: We do not collect your name, contact details, location, or any identifying information.
- **Content Data**: The images you process exist only within your device's memory and local temporary storage; they are never uploaded to any server.
- **Behavioral Data**: We do not monitor your usage habits or interaction patterns.

## 3. Network Permissions
This application **does not request or use network permissions** (`INTERNET`). The technical architecture inherently prevents the possibility of data leakage.

## 4. Permission Transparency
Permissions requested by the app are used solely for local functionality. Users can choose to grant them based on their needs:
- **Storage Access (SAF/PhotoPicker)**: We prioritize system-level secure pickers, allowing image reading without granting the app full storage permissions.
- **Media Read Permission**: Only requested if the user chooses the "Embedded Picker" to scan the local media library.

## 5. Local Storage
The app only stores necessary configuration information locally (e.g., theme preferences, stitching parameters). This data is stored in the app's private sandbox and cannot be accessed by other applications.

## 6. Source Code Transparency
As an open-source project, you can audit our source code at any time to verify the authenticity of the above statements.

## 7. Contact and Feedback
While the app does not collect private data, if you have any questions regarding privacy matters, you may contact us via GitHub Issues.
